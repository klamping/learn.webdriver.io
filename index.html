UI Testing Doesn't Have to be so Hard

Manually testing your website on every change is time consuming and error-prone, and not every project has the budget for QA resources. What if we could easily write automated UI tests in the language that we all know and love: JavaScript!

UI testing has usually been left to the QA team to implement. Dealing with Selenium and Java dependencies are the antithesis of the front-end world. Luckily, WebdriverIO has made automated testing much friendlier for front-end developers.

This course dives deep in to WebdriverIO, an open-source JavaScript library used for Selenium testing. We'll cover all the steps to get started writing automated UI tests, so that you can write and release code with confidence.

12 Lessons to Get You Up and Running:

Lesson One

Your First Test

In only five minutes, we write a simple regression test for our website, validating that users can enter an email in to a form and submit it.

Lesson Two

The WDIO Test Runner

We're all about making things easier; luckily the WDIO test runner sets us up for a common configuration for all of our tests.

Lesson Three

Easy Error-handling with Assertions

Automation is all about making our job easier. Catching errors with assertions allows us to spot failures without work.

Lesson Four

Pause, Debug and other Useful Commands

Sometimes you just need a little break before moving ahead. WebdriverIO provides several utility methods to help with our test cycle.

Lesson Five

Avoid Rework with "execute" and Custom Commands

You'll often find yourself running the same set of commands for specific functionality. Let's make it easier by writing our own custom WebdriverIO Commands.

Lesson Six

Page Objects

Take test organization to the next level by implementing Page Objects, a testing pattern that helps promote reusability and readability.

Lesson Seven

Hooking in to Browserstack & Sauce Labs

Cloud Selenium Services have several great advantages, including loads of browsers and systems to test on.

Lesson Eight

Multibrowser Testing

You put a lot of work in so far, now take full advantage of your tests by running them in a multitude of browsers.

Lesson Nine

Integrating with CICD Systems

Services like TravisCI and Jenkins run your tests for you with every change, and getting them set up is easier than you think.

Lesson Ten

Visual Regression Testing with WebdriverCSS

Sometimes what you're testing is more visual than functional. With WebdriverCSS, we can compare the look of our website between updates, ensuring your design stays right.

Lesson Eleven

Test Reporters

Get important data out of your test results with reports like Allure. This lesson walks through the different reporters available, and how to integrate them in to your CICD system.

Lesson Twelve

Common Tests + Bonus Starter Kit

We wrap up our lessons with a look at several common test scenarios, walking through each one to gain an in-depth understanding of how real-world tests look and work.
